# First Week PM Cadence Workflow

Purpose: Give a new PM a practical first-week operating rhythm for intake, research, synthesis, map generation, and leadership-ready communication.

## Inputs

- inbox items
- PM role context
- active product or workspace context
- optional `workflow_state`
- optional inbound `handoff_contract`

## Outputs

- prioritized Day-1 research starter pack
- one research synthesis artifact
- one visual map
- one leadership or founder-ready update
- updated `workflow_state`
- downstream `handoff_contract` payloads for research, strategy, or communication follow-through

## Runtime control

- maintain visible progress across inbox normalization, research, synthesis, map generation, and first communication output
- record targeted PM questions instead of letting first-week uncertainty turn into generic output
- keep the workflow in `review_needed` until the PM approves the first external-facing update

## Handoffs

- emit `handoff_contract` when the first-week cadence routes work into deeper research, prototype validation, or recurring communication workflows
- preserve the rationale for why each downstream path is recommended

## Rules

- day one should bias toward understanding and synthesis, not premature commitment
- start with inbox normalization, note-card creation, and one fast landscape review
- produce at least one visual map that clarifies the current product or operating state
- recommend the smallest next artifact that helps the PM make or frame a real decision
