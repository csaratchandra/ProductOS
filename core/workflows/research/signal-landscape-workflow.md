# Signal Landscape Workflow

Purpose: Produce a `signal_landscape` that clusters signals across customer, market, competitor, product, support, delivery, stakeholder, and ecosystem sources.

## Inputs

- source note cards
- weak-signal inputs
- optional existing `signal_hypothesis_brief`

## Outputs

- `signal_landscape`

## Rules

- separate noise from watch, emerging pattern, and decision-relevant signals
- cluster signals by class before summarizing them
- preserve signal strength and evidence references for each cluster
- do not let one loud source dominate the watchlist without corroboration
