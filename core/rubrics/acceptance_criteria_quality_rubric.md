# Acceptance Criteria Quality Rubric

Purpose: Judge whether acceptance criteria are specific enough for implementation and testing.

## Dimensions

### 1. Testability

- `strong`: every criterion can be checked objectively
- `acceptable`: most criteria are testable but a few need sharpening
- `weak`: criteria are subjective or unverifiable

### 2. Completeness

- `strong`: covers the important success conditions and key constraints
- `acceptable`: covers the main path but misses some edge or failure conditions
- `weak`: omits major behavior or constraints

### 3. User Relevance

- `strong`: clearly connects to user value or workflow outcome
- `acceptable`: loosely connected to the user or workflow
- `weak`: implementation-shaped but not outcome-shaped

### 4. Scope Precision

- `strong`: clearly distinguishes what is in scope and what is not
- `acceptable`: mostly precise, with small ambiguity
- `weak`: invites scope drift or conflicting interpretations
