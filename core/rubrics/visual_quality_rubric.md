# Visual Quality Rubric

Purpose: Evaluate ProductOS visuals for clarity, narrative discipline, operating usefulness, and rendering integrity across slide, memo, dashboard, and customer-safe forms.

## Dimensions

- narrative clarity: the main message is visible within seconds
- source fidelity: the visual does not overstate certainty or distort the source artifact
- operating utility: decisions, risks, dependencies, or workflow state are easy to act on
- visual hierarchy: emphasis, grouping, and whitespace make the reading path obvious
- comparison quality: trends, deltas, and priorities are interpretable without narration
- audience fit: detail level and framing match PM, leadership, or customer-safe context

## Ratings

- `excellent`: the visual is immediately understandable, decision-oriented, and faithful to source evidence
- `acceptable`: the visual communicates the core message, but hierarchy, labeling, or comparison cues need refinement
- `weak`: the visual is visually present but operationally ambiguous, noisy, or misleading
- `fail`: the visual hides risk, distorts evidence, or cannot support the intended decision

## Minimum Publish Bar

- one primary message per visual
- explicit labels for risk, certainty, or status states when relevant
- clear indication of blocked points, ownership, or dependencies for operational visuals
- audience-safe filtering for customer-facing renderings
- visible linkage back to the source artifact or evidence note
