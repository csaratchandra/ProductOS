# Mastery And Improvement Model

Purpose: Define how ProductOS helps PM capability compound over time and how the system records improvements to both PM behavior and ProductOS Core itself.

## PM Improvement Loop

Each workspace should capture signals such as:

- where the PM accepted coaching
- where the PM rejected coaching
- recurring weak patterns
- recurring strong patterns
- time-to-improvement across repeated tasks
- artifact-quality trend over time

## Reliability Improvement Loop

Each workspace should also track:

- repeated low-confidence outputs
- repeated blocked workflows
- stale-evidence events
- contradictory-source events
- publish-block incidents
- PM reliability overrides

## Integration And Compliance Improvement Loop

Each workspace should also capture:

- integration failures
- stale sync incidents
- source conflict incidents
- privacy review findings
- policy review findings
- compliance review findings

## Mastery Cadence

ProductOS should support:

- weekly coaching review
- monthly skills-gap review
- quarterly PM growth review

These rhythms should produce:

- coaching summary
- strength and weakness profile
- recommended capability focus areas
- evidence of improvement or stagnation
