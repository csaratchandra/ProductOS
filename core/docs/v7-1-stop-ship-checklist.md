# V7.1 Stop-Ship Checklist

Purpose: Define the issues that block the next external ProductOS release.

Release is blocked if any item below is still true.

- the CLI control surface does not run in the supported Python environment
- the frozen eval suite is not `passed`
- release truthfulness is not `healthy`
- discover scoring still mixes generated bundle narration with live decision artifacts
- weekly PM autopilot still depends on seeded or example operating artifacts
- customer workspaces still receive internal, example, or self-hosting residue by default
- the docs or deck path still requires material PM reconstruction before use
- the research loop cannot distinguish `observed`, `inferred`, and `hypothesis`
- the portfolio review and promotion gate disagree on release readiness
- benchmark evidence is absent for speed to insight, rewrite rate, output quality, or operating reliability
- the self-improvement loop still acts as the final promotion judge for the same runtime bundle it assembles
- customer-facing claims exceed what the measured artifact evidence supports

## Required release evidence

- one clean adopted customer workspace
- one persisted discover run
- one persisted operate run
- one docs and deck review package
- one frozen eval report with zero regressions
- one manual release review against the published claim set
