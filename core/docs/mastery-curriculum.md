# PM Mastery Curriculum

Purpose: Define how ProductOS turns repeated workflow evidence into a compounding PM development program rather than isolated coaching comments.

## Curriculum Tracks

- judgment: tradeoff quality, sequencing, and decision rationale
- evidence discipline: traceability, source quality, and confidence handling
- communication: status clarity, leadership framing, and meeting effectiveness
- execution: ownership discipline, dependency visibility, and follow-through
- systems thinking: portfolio awareness, integration realism, and cross-team coordination

## Learning Loop

- detect repeated weak and strong patterns
- map the pattern to a named curriculum track
- recommend a specific next practice
- review whether the practice changed later artifact or workflow quality
- escalate to a higher-intensity coaching path when the same issue repeats without improvement

## Cadence

- weekly: tactical coaching on current operating habits
- monthly: skills-gap and benchmark review
- quarterly: deeper growth review with benchmark calibration and preserved strengths
