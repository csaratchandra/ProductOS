# V7.1 Scope Brief

Purpose: Define the exact customer-facing scope for the next external ProductOS release.

## Version

Recommended version: `V7.1.0`

Use `V8.0.0` only if ProductOS is intentionally repositioned around a new operating model rather than a bounded capability release.

## Release name

`ProductOS V7.1.0: PM Superpower Core`

## Promise

ProductOS gives PMs one governed path from messy inputs to decision-ready product definition, research-backed refinement, stakeholder-ready communication, and a live weekly operating rhythm.

## What ships

- messy input to persisted problem brief, concept brief, and PRD package
- product definition from local source corpus plus governed external research
- PRD to readable docs and stakeholder deck through an evidence-aware presentation path
- weekly PM autopilot from live queues, issues, and release state
- release scoring and promotion gates that reflect actual evidence rather than optimistic narration

## What does not ship

- autonomous PM replacement claims
- open-ended agent swarm claims
- generalized market intelligence for every product question
- universal support for every PM workflow
- broad external publishing and distribution automation

## Target user

A PM or product lead who starts from messy notes, transcripts, research fragments, and scattered decisions and needs a same-day output that is reviewable and traceable.

## Customer-facing claim

Use:

`ProductOS gives PMs a governed operating system for turning messy product inputs into decision-ready outputs.`

Do not use:

`ProductOS gives PMs unlimited autonomous god-mode across all product work.`

## Success metrics

- time from messy input to reviewable decision package
- rewrite rate before PM use
- percentage of important claims with traceable evidence
- percentage of weekly autopilot outputs accepted with minor or no edits
- blocked publish or stale evidence incidents
- release-gate pass rate
