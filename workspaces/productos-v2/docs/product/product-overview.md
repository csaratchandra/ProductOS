## What ProductOS Is

ProductOS is a product operating system for product managers.

It combines structured product state, readable decision documents, governed communication, and bounded AI-agent workflows so PMs can move from noisy inputs to aligned execution with less manual reconstruction.

## Who It Is For

ProductOS is for PMs who need help with:

- turning fragmented signals into decision-ready problem framing
- keeping strategy, roadmap, and PRD work consistent
- creating stakeholder-safe docs and communication quickly
- closing feedback loops into bounded improvements instead of recurring churn

## Problem It Solves

Most PM work breaks when the same truth has to be reconstructed repeatedly across notes, JSON, decks, status updates, and planning discussions.

ProductOS reduces that reconstruction burden by keeping structured state and readable outputs linked through one governed operating model.

## How It Works

ProductOS runs through a small set of golden loops:

- signal to product decision
- decision to stakeholder alignment
- feedback to accepted improvement

Each loop uses artifacts, readable docs, validation lanes, and explicit PM approval instead of one opaque AI-agent route.

## Why It Is Different

ProductOS is not only a note-taking tool, a planning board, or a slide generator.

It is differentiated by:

- structured product artifacts that remain machine-readable
- readable docs that stay linked to source truth
- presentation and communication outputs that are governed rather than improvised
- Ralph-loop quality control for bounded high-stakes slices
- self-hosting evidence that the system is used to improve itself

## Current Release Reality

The current stable line is ProductOS `V6.0.0`.

The current strong surfaces are:

- lifecycle traceability through `release_readiness` in both the self-hosting and starter workspaces
- repo-first discover-to-PRD automation from live inbox inputs
- the `productos` CLI as the canonical next-version control surface
- the verified thin multi-agent adapter layer across Codex, Claude-style, Windsurf, and Antigravity repo-contract paths
- governed readable product docs and live product messaging
- the aligned docs-and-deck stakeholder package
- the supervised weekly PM autopilot
- the feature-scoring and self-improvement refresh loop
- agentic market intelligence with unattended-refresh proof and a leadership-ready distribution path
- Ralph-loop release discipline

The next bounded expansion candidates are lifecycle traceability beyond `release_readiness`, plus external publication adapters and broader distribution packaging beyond the promoted internal leadership path.
