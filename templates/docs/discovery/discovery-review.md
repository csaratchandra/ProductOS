For prototypes, treat detail and interactivity as conditional investments rather than defaults:

- escalate to a high-fidelity interactive prototype only when usability, trust, or workflow-fit ambiguity is still material
- make the uncertainty, quality target, and primary test scenario explicit in `prototype_record.json`
- capture concrete improvement actions on every prototype pass instead of ending discovery with a success-only summary
- require `ux_design_review.json` before a high-fidelity prototype is treated as PRD-ready
- use the prototype-quality rubric to separate "credible v1 candidate" from "useful but still rough"

The intended learning outcome is simple: one PM should be able to explain what happened from discovery through bounded release readiness without reconstructing the story from scattered notes.
